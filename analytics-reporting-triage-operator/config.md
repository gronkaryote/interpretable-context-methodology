## Operator Behavior Overrides

These settings change how the operator applies its default rules.

```
require_ticket_before_sprint: true
  # true | false
  # If true: informal-channel requests (Slack, email) cannot be assigned P1 or P2
  # until a ticket is created. Matches Rule 7.1.

default_access_scope: item-level
  # item-level | workspace-level
  # If item-level: access requests default to the narrowest grant (Rule 5.1).

urgency_validation: strict
  # strict | lenient
  # strict: requester urgency claims must be validated against a concrete event.
  # lenient: requester urgency is taken at face value (not recommended).

logic_duplication_policy: reject
  # reject | escalate | accept
  # reject: requests to rebuild source-system logic in BI are rejected (Rule 2.1 default).
  # escalate: flag for lead review instead of auto-rejecting.
```
