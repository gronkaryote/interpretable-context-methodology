# Archive Backlog

Closed, completed, rejected, and abandoned items. This file is NOT read during triage — it is only consulted when you explicitly ask for historical analysis.

Items are moved here automatically when you run `/close [ID]`.

---

## How to Query This Archive

Ask the operator directly. Examples:

- `"How many P1 requests did we receive last quarter?"`
- `"What percentage of new report requests were rejected for feasibility?"`
- `"Show me all items that were blocked on Data Engineering"`
- `"What's our average time from Triaged to In Sprint for P2 items?"`

The operator will scan this file and return a summary.

## Final Status Definitions

| Final Status | Meaning |
|-------------|---------|
| `Completed` | Work was done and delivered |
| `Rejected` | Request was declined — routing, logic duplication, self-service redirect, or policy |
| `Withdrawn` | Requester cancelled the request |
| `Superseded` | A different request or initiative covered this need |
| `Deferred-Closed` | Was deferred; condition never met; explicitly closed without completing |
| `Duplicate` | Identical or substantially similar request already exists or was completed |
